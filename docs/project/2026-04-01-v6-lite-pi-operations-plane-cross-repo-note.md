# V6-lite Pi operations plane — cross-repo boundary note

## Why this note exists

A real next-wave concern is now visible across repos:

- Pi host lifecycle is simplifying around enriched `session_start` metadata.
- The live TUI currently mixes useful memory with too much persistent vertical noise.
- Subagent history should stay visible enough to be useful, but not as an unbounded always-on backlog.
- A durable timeline for Pi lifecycle and subagent milestones now has an obvious candidate home in `softwareco/infra/hooks-logging`.

That does **not** yet justify a shadow graph or a rewrite of Pi's native session tree.
It does justify one bounded cross-repo wave.

## Bounded direction

For the V6-lite wave:

- **Pi native session state remains the live authority** for current conversation structure and `/tree` semantics.
- **`softwareco/owned/pi-extensions` owns the live operator surface** for low-clutter, session-aware runtime visibility.
- **`softwareco/infra/hooks-logging` owns the durable milestone ledger** for replayable Pi lifecycle/subagent events.
- **`softwareco/owned/agent-kernel` owns the executable decision/fanout coordination spine** for the cross-repo wave.
- **`holdingco/governance-kernel` FCOS remains the shared concern/authority boundary** for the wave as a whole.

## What this wave should not do

Do **not** use this wave to claim that:

- subagents are now first-class replacements for Pi session branches,
- hooks-logging is now the live authority for current Pi session state,
- the monorepo root should absorb package-local runtime ownership,
- or a semantic/shadow graph is already required before a bounded replayable milestone ledger exists.

## Smallest truthful next move

The smallest truthful move is:

1. converge extension lifecycle handling onto enriched `session_start` metadata,
2. keep the live Pi surface session-aware and low-clutter,
3. define one bounded Pi milestone ledger slice in hooks-logging,
4. coordinate the wave through AK fanout backed by one FCOS concern.

That is enough to test the V6-lite idea honestly.
It is not yet the full V6 architecture.
